# mind-gateway 代码编写规范

这份文件用于固定 `mind-gateway` 的代码风格要求。  
后续所有新增或修改代码，默认都遵循这里的规则。

## 1. 注释语言

- 所有注释统一使用中文。
- 所有日志输出统一使用中文。

## 2. 函数注释规则

- 每个函数都必须写注释。
- 函数注释使用简洁格式：

```go
// 函数名 注释内容
```

示例：

```go
// NewApp 创建应用实例并装配运行所需依赖
func NewApp() *App {
    ...
}
```

## 3. 大函数内部注释规则

- 如果一个函数内部包含多个明显步骤，则每一步前都要写注释。
- 注释重点说明“这一步在做什么”，不要求写成长段说明。

示例：

```go
// HandleChat 处理聊天补全请求
func HandleChat(...) {
    // 解析并校验请求参数
    ...

    // 构造内部请求对象
    ...

    // 调用网关核心执行链路
    ...

    // 将结果编码为 OpenAI 兼容响应
    ...
}
```

## 4. 结构体与字段注释规则

- 每个结构体都应写注释。
- 结构体中的每个字段都应写注释。
- 字段注释紧跟在字段定义后面，说明该字段的职责或含义。

示例：

```go
// ServerConfig 定义 HTTP 服务启动配置
type ServerConfig struct {
    Host string // 服务监听地址
    Port int    // 服务监听端口
}
```

## 5. 日志规则

- 所有程序日志统一使用中文。
- 日志内容应直接说明当前动作、结果或错误原因。
- 避免使用模糊英文短语作为主日志内容。

示例：

```go
log.Printf("启动 HTTP 服务，监听地址: %s", addr)
log.Printf("加载配置失败: %v", err)
```

## 6. 执行要求

- 新增函数时，先写注释，再写实现。
- 修改已有函数时，如果注释缺失，需要一并补齐。
- 新增结构体字段时，必须同步补字段注释。
- 拆分大函数时，保留步骤注释，不允许全部删除。

## 7. Golang 文件与目录命名规则

- Go 项目的目录名称优先使用单数，而不是复数。
- 例如使用 `schema`，不要使用 `schemas`。
- 例如使用 `handler`，不要使用 `handlers`。
- Go 文件名与目录名统一使用 `snake_case`。
- 不使用驼峰目录名。
- 不使用混合风格文件名。

示例：

- `transport/http/handler`
- `provider/openai`
- `request_log.go`
- `health_test.go`
